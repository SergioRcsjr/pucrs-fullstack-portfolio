# 🚨Estudo-de-Qualidade-e-Teste-de-Software🚨

# Sobre a Disciplina

Introdução aos conceitos de testes unitários, de integração e de UI, além de garantia de qualidade de software e estudo de métricas para controle de qualidade no desenvolvimento de software.

# Conceitos chave

- Testes unitários: Verificam funções isoladas, com stubs e esqueletos para isolamento, estratégias como TDD (codificar-testar-projetar) e BDD (foco em componente isolado)  
- Testes de integração: Avaliam o ecossistema onde métodos se integram, rodando localmente, em pipeline ou a cada commit  
- Testes de UI: Focam em alinhamento, tipografia, iconografia, ancoragem e vocabulário na interface, com equipes de qualidade, dev e design colaborando  
- Garantia de qualidade: Cobertura de requisitos funcionais e não funcionais, processos de build (commit/validação/teste completo), review e merge com camada de qualidade  
- Métricas de qualidade: Impactos na qualidade e tempo de desenvolvimento, independência entre testes, feedback rápido para evitar bugs  
- Boas práticas: Identificar unidade como parte indivisível do código, consistência visual na experiência do usuário, ecossistema front-end/back-end  

# Ferramentas iniciais

Ferramentas para testes unitários/integração/UI (ex: stubs/esqueletos para isolamento), pipelines para integração contínua, frameworks como JUnit/Jest para TDD/BDD
