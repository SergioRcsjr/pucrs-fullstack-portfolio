# 🚨Estudo-de-Segurança-de-Software🚨

# Sobre a Disciplina

Introdução à segurança de software, com foco em métodos de criptografia para transmissão e armazenamento de dados, protocolos seguros, vulnerabilidades comuns e boas práticas para desenvolvimento seguro.

# Conceitos chave

- Segurança de software: Proteção contra ameaças como hackers, vazamento de dados, phishing e malware  
- Criptografia simétrica: Usa chave única para cifrar e decifrar (ex: AES-256)  
- Criptografia assimétrica: Usa chaves pública/privada (ex: RSA, Diffie-Hellman)  
- Protocolos seguros  
  HTTPS/SSL/TLS para navegadores  
  LibSignal para apps de conversa  
- OWASP Top 10: Vulnerabilidades como injeção, autenticação quebrada, exposição de dados sensíveis  
- Autenticação e autorização: Métodos seguros, análise de riscos, conformidade com LGPD  
- Boas práticas: Análise estática/dinâmica, testes de invasão, gerenciamento de chaves

# Ferramentas iniciais

OWASP tools, Sonar (análise de código), algoritmos AES/RSA/SHA-256, protocolos HTTPS/LibSignal
