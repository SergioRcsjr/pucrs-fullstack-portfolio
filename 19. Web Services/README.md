# 🚨Estudo-de-Web-Services🚨

# Sobre a Disciplina

Estudo sobre web services, com foco em conceitos de arquitetura monolítica, revisão de SOAP, REST, GraphQL e descritores de serviços, soluções serverless e construção de soluções com frameworks REST e GraphQL.

# Conceitos chave

- Web services: Serviços independentes para comunicação entre sistemas, evoluindo da internet (ARPAnet) para nuvem  
- Arquitetura monolítica: App única com todas funcionalidades acopladas  
- SOAP: Protocolo baseado em XML para troca de mensagens estruturadas  
- REST: Estilo arquitetural com HTTP, recursos como URIs, stateless  
- GraphQL: Linguagem de consulta para APIs, permitindo queries precisas vs REST over/under-fetching  
- Serverless: Execução sem gerenciar servidores, foco em funções (FaaS)  
- Twelve Factor App: Princípios para apps escaláveis: base única de código, dependências explícitas, config via ambiente, backing services, build/release/run separados  

# Ferramentas iniciais

Frameworks REST (Node.js/Express), GraphQL (Apollo), descritores (OpenAPI, WSDL), ferramentas W3C, testes unitários
