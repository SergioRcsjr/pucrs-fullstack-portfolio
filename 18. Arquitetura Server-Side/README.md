# 🚨Estudo-de-Arquitetura-Server-Side🚨

# Sobre a Disciplina

Estudo sobre arquitetura client-server para aplicações web, com foco em frameworks MVC server-side, programação assíncrona e reativa, desenvolvimento de apps web usando serviços e boas práticas como testes unitários e validação.

# Conceitos chave

- Arquitetura client-server: Estrutura para apps web com front-end e back-end se comunicando  
- Frameworks MVC server-side: Node.js, Express, NestJS para organizar código em Model, View, Controller  
- Programação assíncrona: Gerencia tarefas simultâneas sem bloquear o fluxo  
- Programação reativa: Baseada em fluxos de dados assíncronos, fundamentos: elasticidade, message drive, responsividade e resiliência  
- Microserviços vs monolitos: Serviços independentes (escaláveis, APIs definidas) vs tudo acoplado em um único app  
- Testes unitários: Verificam funções isoladas, dão feedback rápido e evitam bugs  
- Validação: Usar ferramentas como Yup para esquemas e verificação de dados  

# Ferramentas iniciais

Node.js, Express, NestJS, Yup (validação), ferramentas de testes unitários (ex: Jest)
